## Documentation Structure

This documentation is organized into four main sections covering architecture, development, legal considerations, and future roadmap.

---

## Architecture Documentation

Detailed technical specifications of the protocol's design and implementation.

### [System Overview](./architecture/OVERVIEW.md)
High-level architecture, design principles, system components, and interaction patterns between protocol layers.

### [Smart Contract Specifications](./architecture/SMART-CONTRACTS.md)
Complete specifications for all eight core contracts including interfaces, state variables, functions, events, and access control mechanisms.

### [Security Mechanisms](./architecture/SECURITY.md)
Security features, threat models, attack mitigation strategies, audit procedures, and best practices for safe deployment.

---

## Development Documentation

Practical guides for developers working with or integrating the protocol.

### [Development Timeline](./development/TIMELINE.md)
15-month development roadmap with detailed milestones, deliverables, and dependencies across three major phases.

### [Testing Strategy](./development/TESTING.md)
Comprehensive testing methodology including unit tests, integration tests, security tests, and coverage requirements.

### [Deployment Guide](./development/DEPLOYMENTS.md)
Step-by-step deployment procedures for testnet and mainnet across Ethereum, BSC, and Polygon networks.

## Legal and Compliance

Business structure, regulatory considerations, and compliance requirements.

### [Corporate Structure](./legal/STRUCTURE.md)
Multi-phase business entity strategy covering Bulgarian startup phase, Lithuanian expansion, intellectual property protection, and co-founder integration.

### [Compliance Requirements](./legal/COMPLIANCE.md)
Regulatory analysis, licensing considerations, data protection requirements, and terms of service framework.

## Future Development

Long-term expansion plans and community governance framework.

### [Fiat Payment Integration](./future/FIAT-INTEGRATION.md)
Technical architecture and implementation plan for Stripe and PayPal integration using merchant-direct API aggregation model.

### [Governance System](./future/GOVERNANCE.md)
Community-driven protocol upgrade mechanism including proposal submission, voting procedures, and implementation processes.
